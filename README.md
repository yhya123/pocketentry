# دفتر محاسبي بسيط

تطبيق Flutter لإدارة الحسابات الحسابية والعمليات المالية — واجهة عربية كاملة (RTL)، قاعدة بيانات SQLite محلية، وإدارة حالة عبر GetX.

## التشغيل

```bash
flutter pub get
flutter run
```

**Android:** يعمل مباشرة بعد تثبيت الحزم.  
**المتطلبات:** Flutter SDK ^3.11، جهاز Android أو محاكي.

## البنية (Clean Architecture + MVC)

```
lib/
├── app/                 # تهيئة التطبيق
├── bindings/            # حقن التبعيات (GetX)
├── core/                # ثوابت، ثيم، أدوات، ويدجتات مشتركة
├── data/
│   ├── database/        # SQLite helper
│   ├── datasources/     # مصادر البيانات المحلية
│   ├── models/          # نماذج البيانات (DTO)
│   └── repositories/    # تنفيذ المستودعات
├── domain/
│   ├── entities/        # كيانات الأعمال
│   ├── repositories/    # واجهات المستودعات
│   └── usecases/        # حالات الاستخدام
├── presentation/
│   ├── controllers/     # GetX controllers (MVC)
│   ├── views/           # الشاشات
│   └── widgets/         # مكونات UI
├── routes/              # التنقل
└── services/            # PDF، نسخ احتياطي، صور
```

## قرارات تصميمية

| القرار | التفاصيل |
|--------|----------|
| **نوع العملية** | `debit` (عليه) يزيد المديونية، `credit` (له) يقللها. الرصيد = عليه − له |
| **العملات** | ريال يمني، ريال سعودي، دولار — رصيد منفصل لكل عملة |
| **الصور** | تُحفظ محلياً في مجلد `receipts` داخل documents |
| **النسخ الاحتياطي** | واجهة `BackupProvider` — محلي يعمل، Google Drive جاهز للتوسعة |
| **PDF** | `pdf` + `printing` + `share_plus` مع خط عربي Noto Naskh |

## الشاشات

- الرئيسية (إجراءات سريعة: إضافة عملية / إضافة حساب)
- **إضافة عملية مباشرة** — اختيار الحساب بـ Auto Complete أو إنشاؤه من نفس الشاشة
- **الحسابات** — قائمة مركزية لجميع الحسابات (بحث، فلترة، أرصدة، آخر حركة)
- التصنيفات → الحسابات → تفاصيل الحساب → العمليات (المسار السابق ما زال يعمل)
- إضافة/تعديل عملية (مع صورة السند)
- التقارير (5 أنواع + تصفية + PDF/طباعة/مشاركة)
- الإعدادات، النسخ الاحتياطي، الدعم، البحث

### تدفق الاستخدام اليومي (UX)

1. فتح التطبيق → **إضافة عملية**
2. كتابة اسم الحساب → اختيار من الاقتراحات أو **إضافة حساب جديد** (Bottom Sheet)
3. إدخال المبلغ والتفاصيل → حفظ

لا تغيير في منطق قاعدة البيانات أو التقارير؛ التحسينات في طبقة العرض والتنقل فقط. إعدادات `last_used_currency` و`recent_person_ids` تُخزَّن في جدول `settings` الحالي.

## التوسعة المستقبلية

- تفعيل `GoogleDriveBackupProvider` عبر `google_sign_in` + Drive API
- مزامنة سحابية / تعدد مستخدمين عبر طبقة `domain/repositories` دون تغيير UI
- دمج داخل نظام أكبر: استبدال `data/datasources` بـ API remote

## الجداول (SQLite)

- `categories` — التصنيفات (افتراضي: «عام»)
- `persons` — الحسابات
- `transactions` — العمليات المالية
- `settings` — key/value للإعدادات
